# سياسة رسائل الغرف والخاص

هذه السياسة قرار معماري ثابت في المشروع الحالي.

## القرار

- رسائل الغرف لا تخزن في قاعدة البيانات.
- رسائل المحادثة الخاصة لا تخزن في قاعدة البيانات.
- الرسائل تكون realtime ومؤقتة فقط أثناء تشغيل السيرفر واتصال المستخدمين.
- الهدف هو جعل الإرسال والاستقبال سريعاً جداً وتقليل الضغط على قاعدة البيانات.

## ماذا يعني ذلك؟

- عند إعادة تشغيل السيرفر، لا نضمن بقاء سجل رسائل الغرف أو الخاص.
- لا يوجد أرشيف دائم لكل المحادثات.
- قاعدة البيانات تستخدم للحسابات، الغرف، الصلاحيات، الإعدادات، المحافظ، الوكلاء، الهدايا، البلاغات، وسجلات الإدارة.
- الحسابات تحفظ في `users`، وخصائص العضو المرئية تحفظ في `user_profiles`.
- الرسائل اللحظية تستخدم ذاكرة السيرفر في التطوير، ثم Redis أو طبقة ذاكرة موزعة في الإنتاج.

## الاستثناءات المسموحة

يسمح بتخزين بيانات مرتبطة بالرسائل فقط في حالات محددة:

- بلاغ على رسالة مسيئة.
- رسالة محذوفة تحتاج مراجعة إدارية.
- أمر إداري أو حدث مراقبة.
- رسالة مثبتة إذا قررت الإدارة أنها محتوى دائم.
- إحصاءات عامة بدون تخزين نصوص المحادثات.

## مسار الإنتاج المقترح

في الإنتاج الكبير:

1. الواجهة ترسل الرسالة إلى السيرفر عبر WebSocket أو API سريع.
2. السيرفر يتحقق من الجلسة والصلاحيات وحجم الرسالة.
3. الرسالة تبث فوراً للموجودين في الغرفة أو المحادثة الخاصة.
4. Redis يستخدم كطبقة توزيع مؤقتة بين السيرفرات.
5. لا يتم إدخال نص الرسالة في PostgreSQL إلا إذا كانت من الاستثناءات أعلاه.
